Submitting programs

How do I submit a program?

To submit a program you wrote for an assignment:
  • Navigate to the assignment.
  • Select the file in the "submit" box.
  • Click on "submit".

Programs with multiple files

For some assignments you will create more than one source file. All files should be submitted to Justitia.

You can also bundle multiple files and directories in a ZIP archive, and submit that instead.

How are programs judged?

Justitia judges submitted programs by compiling them, and then executing them with a predefined testset.

After a submitted program has been judged by Justitia, you will get feedback. The amount of feedback depends on the settings by the course administrator.

The possible status codes are:
Pending The submission has been received, but is not yet being judged.
Judging The submission is currently being compiled or run.
Passed The submitted program produced the correct answer.
Failed: unknown language There are no source files in the submission, at least not in a language that Justitia understands.
Failed: compile error Your submission failed to compile. The compiler error message will be available if you click on the "view error message" link.
Failed: runtime error There was a runtime error when the program was executed, for example "out of memory", "division by zero" or "NullPointerException".
Failed: wrong answer Your answer did not match that of the reference implementation.
Missed deadline Sometimes it is allowed to submit files after the deadline. The status 'Missed deadline' indicates that the program provided the correct answer, but was submitted after the deadline.

Why can't I find my lab partner?

You can only add users to a group if they are known by Justitia. If Justitia is configured to use LDAP for authentication, make sure your lab partner has logged in with Justitia at least once.
